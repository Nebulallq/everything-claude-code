# 更新日志 - Human-in-the-Loop 自动化研发协议

## 更新时间
2025-03-11

## 更新概述
对《单 Agent + 人工确认流程》文档进行全面整合与增强，参考 ECC 项目现有 agents/commands 模块，补充了各阶段的输出模板、检查清单和执行流程。

---

## 主要变更

### 一、执行流程调整
- 原 5 阶段流程扩展为 **6 阶段**
- 新增「阶段6：输出开发报告」
- 流程图更新：需求理解 → 技术设计 → 代码实现 → 测试补充 → 综合自检 → **输出报告**

---

### 二、各阶段内容增强

#### 阶段1：需求理解
- [保持原有模板]
- 功能目标、业务规则、边界条件、异常场景、影响范围、不确定点

#### 阶段2：技术设计 ✨ 新增模板
- **设计方案**：核心改动、涉及文件、新增/修改类/方法
- **风险评估**：HIGH/MEDIUM/LOW 三级风险分类
- **依赖检查**：依赖库、数据库、配置、API 变更
- **兼容性影响**：破坏性变更、向后兼容性、回滚方案

#### 阶段3：代码实现 ✨ 新增模板
- **文件变更清单**：路径、操作类型、说明
- **关键改动摘要**：逐项列出改动
- **自验证结果**：编译、Lint、安全检查

#### 阶段4：测试补充 ✨ 增强
- **TDD 执行循环**：RED → GREEN → REFACTOR → REPEAT
- **执行步骤**：SCAFFOLD → RED → GREEN → REFACTOR → COVERAGE
- **分级覆盖率要求**：
  - 标准代码：80%
  - 财务/认证/安全关键：100%
- **测试类型清单**：单元、边界、错误、集成测试

#### 阶段5：综合自检与验证 ✨ 大幅增强
- **安全问题（CRITICAL）**：
  - 硬编码凭证检查
  - OWASP Top 10 完整清单
- **代码质量（HIGH）**：
  - 函数 ≤ 50 行
  - 文件 ≤ 800 行
  - 嵌套深度 ≤ 4 层
  - 错误处理、调试语句检查
- **最佳实践（MEDIUM）**：不可变模式、复杂度、测试覆盖
- **综合验证流程**：构建 → 类型 → Lint → 测试 → 安全扫描
- **验证报告格式**：结构化输出

#### 阶段6：输出开发报告 🆕 全新
1. **需求概述**：原始需求、理解、确认记录
2. **文件变更清单**：变更表格、统计
3. **关键改动摘要**：逐项说明
4. **测试结果**：覆盖率、执行记录、验证清单
5. **风险评估**：风险等级、破坏性变更、依赖变更
6. **部署建议**：部署前检查、步骤、验证
7. **遗留问题**：待处理事项、技术债务
8. **后续建议**：优化建议、扩展方向、相关文档
